# Repository Architecture

## Purpose

This repository is structured to support a public company platform today and future product or systems layers over time.

## Current Layers

- `src/`: application UI and page composition
- `projects/`: structured project and slate data
- `localization/`: language content separated from UI logic
- `docs/`: investor, project, and technology documentation
- `config/`: shared platform configuration

## Expansion Path

The architecture is intentionally prepared for:

- Q³ documentation and developer content
- AI-assisted production tools
- backend services or internal APIs
- richer project data models
- contributor workflows for design, product, and documentation teams
