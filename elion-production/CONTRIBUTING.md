# Contributing

## Standards

This repository should read like a production company platform maintained for long-term use.

Please follow these expectations:

- write clear, maintainable code
- keep content, configuration, and UI concerns separated
- avoid speculative abstractions with no active use case
- preserve professional tone across code comments, docs, and copy

## Pull Request Checklist

- describe the business or product reason for the change
- keep scope focused
- update docs when content, structure, or setup changes
- verify localization impact when visible copy changes
- avoid committing generated artifacts unless intentionally required

## Design Expectations

- prioritize hierarchy, clarity, and restraint
- do not introduce template-like or trend-driven UI patterns
- keep the repository understandable for new developers within minutes
